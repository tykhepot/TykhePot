# TykhePot 社区激励系统 ($TPOT)

## 概述

通过代币激励，建立一个活跃的开源社区和强大的推广网络。

---

## 二、审核与评估机制

### 1. 贡献追踪系统

#### GitHub集成
```
PR提交 → 自动化评估 → 人工审核 → 确定奖励 → 发放代币
```

**自动化评估维度**:
- 代码行数 (权重: 20%)
- 文件影响数 (权重: 10%)
- 测试覆盖率变化 (权重: 20%)
- 性能提升指标 (权重: 30%)
- 代码质量评分 (权重: 20%)

#### 社区贡献追踪
- 任务认领系统 (类似Gitcoin)
- 贡献者排行榜
- 月度/季度评选

### 2. 多级审核机制

```
┌─────────────┐
│ 贡献者提交   │
└──────┬──────┘
       ↓
┌─────────────┐
│ 初审(自动化) │ ← 基础筛选
└──────┬──────┘
       ↓
┌─────────────┐
│ 技术审核    │ ← 核心开发者
└──────┬──────┘
       ↓
┌─────────────┐
│ 价值评估    │ ← 经济模型团队
└──────┬──────┘
       ↓
┌─────────────┐
│ 社区投票    │ ← 重大贡献(>10K)
└──────┬──────┘
       ↓
┌─────────────┐
│ 发放奖励    │
└─────────────┘
```

### 3. 审核委员会

**技术委员会** (3-5人):
- 核心开发者
- 安全专家
- 技术顾问

**社区委员会** (3-5人):
- 社区经理
- 活跃贡献者
- 生态合作伙伴

## 四、技术实现方案

### 1. 智能合约模块

#### CommunityRewards 合约
```rust
#[program]
pub mod community_rewards {
    // 提交贡献申请
    pub fn submit_contribution(
        ctx: Context<SubmitContribution>,
        contribution_type: ContributionType,
        description: String,
        proof_url: String,
    ) -> Result<()>;
    
    // 审核贡献 (委员会)
    pub fn review_contribution(
        ctx: Context<ReviewContribution>,
        contribution_id: u64,
        status: ReviewStatus,
        reward_amount: u64,
    ) -> Result<()>;
    
    // 社区投票 (重大贡献)
    pub fn vote_contribution(
        ctx: Context<VoteContribution>,
        contribution_id: u64,
        approve: bool,
    ) -> Result<()>;
    
    // 领取奖励
    pub fn claim_reward(
        ctx: Context<ClaimReward>,
        contribution_id: u64,
    ) -> Result<()>;
}
```

#### 数据结构
```rust
#[account]
pub struct Contribution {
    pub contributor: Pubkey,
    pub contribution_type: ContributionType,
    pub description: String,
    pub proof_url: String,
    pub status: ReviewStatus,
    pub reward_amount: u64,
    pub submitted_at: i64,
    pub reviewed_at: Option<i64>,
    pub claimed: bool,
}

#[account]
pub struct CommunityRewardsState {
    pub authority: Pubkey,
    pub developer_pool: u64,
    pub community_pool: u64,
    pub marketing_pool: u64,
    pub committee_members: Vec<Pubkey>,
    pub total_distributed: u64,
}
```

## 五、操作流程

### 开发者贡献流程

```
1. 发现需求/问题
      ↓
2. Fork代码库
      ↓
3. 开发实现
      ↓
4. 提交PR + 奖励申请
      ↓
5. 自动评估 + 人工审核
      ↓
6. 合并代码 → 发放奖励
```

### 社区贡献流程

```
1. 查看任务列表
      ↓
2. 认领任务
      ↓
3. 完成任务
      ↓
4. 提交成果 + 证明
      ↓
5. 社区审核
      ↓
6. 审核通过 → 发放奖励
```
